🔴 [블로그] 판매 zip이 오염된 채 출하 대기 중이었음 — 가장 큰 발견
dist에 있던 판매 zip을 열어보니 패키저가 "제외한다"고 명시한 것들이 그대로 들어 있었습니다:
  • 내부 검수문서 _판매점검\비밀정보_스캔.md 동봉 — 개인 네이버 아이디·개인 폴더 경로가 "어디서 발견돼 지웠는지" 이력이 통째로 적힌 문서. 구매자에게 가면 정화 작업이 도로 무효.
  • 디버그·개인용 스크립트 41개 동봉 (허용목록 8개만 남기게 되어 있었음)
  • 그 결과 npm test가 켜자마자 실패 (개인 원고용 테스트가 없는 파일을 import)
원인: 패키저(배포본_만들기.ps1)가 UTF-8 BOM 없이 저장돼 있어, PowerShell 5.1이 한글 제외 규칙을 깨진 글자로 읽음 → 에러 없이 "완료"라고 뜨면서 제외가 전부 무시됨. 실행해도 잡을 수 없고 zip을 열어봐야만 보이는 유형.
✅ 처리: ps1을 UTF-8 BOM으로 재저장 → zip 재생성 → 재검증(238→210파일, 내부문서 제거, 스크립트 41→14개, npm test 전부 통과, 개인식별자 0건). 오염됐던 구버전 zip은 ...zip.QA3백업_구버전으로 물증 보존. 다른 두 신규 패키저에도 BOM 강제 + "유출 시 중단" 안전검사 내장.
🔴 [유튜브] 자막 폰트 파일이 판매본에 없음
롱폼·쇼츠 템플릿 4종이 전부 assets/fonts/GowunBatang-Bold.ttf(고운바탕 캘리체)를 요구하는데, 판매 사본과 _dist 어디에도 없었음(운영 원본에만 존재). 렌더가 죽지는 않지만 구매자 영상 자막이 전부 밋밋한 시스템 폰트로 조용히 바뀜 — 데모와 다른 결과물이 나가는 결함.
✅ 처리: 원본에서 폰트 복사(고운바탕 = OFL 오픈라이선스, 재배포 가능) + 패키저 화이트리스트 등재 + _dist 재생성 확인.
🟡 [유튜브] requirements.txt에 pillow 누락 — 세로 배경 크롭 도구(make_vertical.py)가 PIL을 쓰는데 설치 목록에 없어, 쓰는 순간 크래시. → 추가 완료.

🟡 [유튜브] 즉사 코드 3개 동봉 — playlist_*.py 3개가 _dist에 없는 모듈(playlist_common)을 import → 실행 즉시 사망. 재생목록 기능은 판매 범위 밖이므로 화이트리스트에서 제외하고 _dist 재생성. → 제거 완료.

🟡 [쓰레드·인스타] 판매 zip 자동화 부재 — "손으로 zip 만들라"는 절차만 있었음. 블로그 사고와 동일한 유형의 사고 대기 상태. → 두 제품 모두 패키저 신설(BOM·안전검사 내장) + zip 생성·검증 완료.
오탐 정정(결함 아님): 유튜브 onset_check.py의 scipy import는 requirements 누락이 아니라 soundfile→scipy→wave 3단 폴백 구조의 2순위였음. soundfile이 설치 목록에 있어 정상 동작.
