1편. 유튜브 채널 현황, 매일 아침 텔레그램으로 자동으로 받는 방법

세 줄 요약. 프로그램 하나가 아침마다 유튜브 구독자·조회수 증감을 정리해 텔레그램으로 보내줍니다. 준비물은 ① 정보 가져오는 창구(API 키) ② 나에게 배달하는 우체부(텔레그램 봇) 두 가지. 이 두 재료면 유튜브뿐 아니라 뉴스·날씨·주가도 매일 자동으로 받을 수 있어요 — 그래서 이 글 뒤에 따라 할 연습 예제도 같이 넣었습니다. 코딩 몰라도, 그대로 따라만 하면 됩니다.

저는 음악 유튜브 채널을 세 개 운영합니다. 아침에 눈 뜨면 제일 먼저 하는 일이 스튜디오 앱을 세 번 여는 거였어요. 어제보다 구독자가 늘었나, 어떤 영상이 조금 튀었나. 별것 아닌데 매일 하려니 은근히 귀찮고, 세 개를 왔다 갔다 하다 보면 헷갈리기도 하고요.

그래서 생각했습니다. 이 반복을 컴퓨터한테 시키면 안 되나? 결론부터 말하면 됐습니다. 지금은 아침 9시에 카톡 대신 텔레그램으로 "채널 3개 현황"이 한 장으로 날아옵니다.

이번 편은 진짜 왕초보 기준으로 씁니다. 컴퓨터 잘 몰라도, 아래 순서를 글자 그대로 따라 하시면 됩니다. 중간에 모르는 단어가 나와도 그때그때 쉽게 풀어드릴게요.

이 글이 진짜 알려주는 것 — '나만의 정보 비서' 재료 2개

본론은 유튜브지만, 사실 여기서 배우는 건 어디에나 쓰는 재료 두 개예요. 이 둘만 알면 유튜브 말고 뉴스·날씨·주가도 매일 자동으로 내 폰에 배달시킬 수 있습니다.

① 정보를 가져오는 창구(API 키) — "야, 그 숫자 좀 알려줘"라고 물어보는 출입증이에요. 유튜브엔 유튜브 열쇠, 날씨엔 날씨 열쇠… 대상만 다르지 원리는 똑같습니다.
② 나에게 배달하는 우체부(텔레그램 봇) — 가져온 정보를 내 폰으로 대신 보내주는 심부름꾼입니다. 한 번 만들면 뭐든 배달시킬 수 있어요.

그래서 이 글은 유튜브를 예로 이 둘을 만들어보고, 뒤에서 뉴스·시세 같은 다른 연습도 같이 해봅니다. 유튜브에 관심 없어도 '재료' 배우러 읽을 가치가 있어요. 자, 하나씩 갑니다.

STEP 1. 유튜브 열쇠(API 키) 받기 — 약 5분

📍 아래를 1번부터 순서대로 그대로 따라 하세요
  1. 구글 클라우드 콘솔에 들어갑니다. 인터넷 주소창에 console.cloud.google.com 을 치고 엔터. 내 유튜브 채널을 쓰는 구글 계정으로 로그인하세요. 처음이면 나라 선택·약관 동의 화면이 한 번 뜨는데, 체크하고 계속 누르면 됩니다. (돈 안 나갑니다. 카드 등록 필요 없어요.)
  2. 새 프로젝트를 하나 만듭니다. '프로젝트'는 그냥 내 작업을 담을 폴더라고 생각하세요. 화면 맨 위 왼쪽에 있는 프로젝트 이름(또는 ▼) 을 누르면 창이 뜹니다 → 오른쪽 위 새 프로젝트 → 이름은 아무거나(예: 내채널알림) → 만들기. 몇 초 뒤 만들어집니다.
  3. 만든 프로젝트가 선택돼 있는지 확인합니다. 다시 맨 위 왼쪽을 봐서, 방금 만든 내채널알림 이 골라져 있으면 됩니다. 아니면 눌러서 그걸로 바꿔주세요. (엉뚱한 폴더에서 작업하면 헷갈립니다.)
  4. 유튜브 기능을 '켭니다'. 왼쪽 위 줄 세 개(≡) 메뉴 → API 및 서비스 › 라이브러리. 가운데 검색창에 YouTube Data API v3 를 치고 엔터 → 나온 결과를 클릭 → 파란색 사용 버튼을 누릅니다. 이게 "유튜브야, 내 프로그램이 너한테 물어봐도 돼"를 켜는 스위치예요.
  5. 드디어 열쇠(API 키)를 만듭니다. 다시 왼쪽 줄 세 개(≡) → API 및 서비스 › 사용자 인증 정보 → 화면 위쪽의 + 사용자 인증 정보 만들기 → 뜨는 목록에서 API 키 를 고릅니다.
  6. 열쇠를 복사해서 메모장에 붙여넣고 저장합니다. 잠깐 뒤에 AIza… 로 시작하는 영어·숫자가 길게 섞인 문자열이 뜹니다. 이게 바로 열쇠예요. 옆의 복사 아이콘을 눌러 복사 → 컴퓨터 메모장에 붙여넣고 저장해 두세요. 잠시 뒤에 씁니다.
⚠️ 이 열쇠는 내 신분증과 같습니다. 남이 보면 내 몫으로 몰래 쓸 수 있어요. 블로그·카톡·오픈채팅에 그대로 올리지 마세요. 메모장에만 조용히 보관.

STEP 2. 텔레그램 봇(우체부) 만들기 — 약 3분

이제 그 숫자를 매일 나한테 보내줄 우체부를 만듭니다. 텔레그램 앱만 있으면 됩니다(없으면 앱스토어에서 '텔레그램' 설치, 전화번호로 가입).

📍 텔레그램 앱을 열고 따라 하세요
  1. 'BotFather'를 찾습니다. 텔레그램 위쪽 돋보기(검색)에 BotFather 를 칩니다. 여러 개 뜨면 이름 옆에 파란 체크(✔ 공식 인증)가 붙은 걸 고르세요. 이게 봇을 만들어주는 공식 봇입니다.
  2. 봇 만들기를 시작합니다. 맨 아래 메시지 입력창에 /newbot 이라고 치고 전송하세요.
  3. 봇 이름을 정합니다. "이름을 뭐로 할래?"라고 물어봅니다. 아무거나 한글로 써도 됩니다(예: 내채널 알림) → 전송.
  4. 봇 아이디를 정합니다. 이번엔 "아이디를 뭐로 할래?"라고 물어봐요. 반드시 끝이 bot 으로 끝나야 합니다(예: mychannel_alert_bot). 이미 있는 이름이면 다른 걸로 바꿔서 다시 보내세요.
  5. 봇 토큰(비밀번호)을 받아 저장합니다. 성공하면 Use this token to access the HTTP API: 라는 문구 밑에 숫자와 영문이 섞인 긴 문자열을 줍니다. 이게 '봇 토큰'이에요. STEP 1의 열쇠처럼 메모장에 붙여넣고 저장. 이것도 비밀입니다.
  6. 내 번호(챗 ID)를 알아냅니다. 우체부가 "누구 집에 배달할지"를 알아야겠죠? 그게 내 챗 ID(고유 번호)입니다. 검색창에 userinfobot 을 쳐서 그 봇에 들어가 시작(START) 을 누르면, 바로 Id: 123456789 처럼 내 번호를 알려줍니다. 그 숫자도 메모장에 저장하세요.

STEP 3. 첫 성공 — 봇이 나한테 진짜로 말 걸게 해보기 (3분)

재료 두 개를 만들었으니, 가장 작은 성공부터 맛봅시다. 거창한 것 말고, 내 봇이 나에게 "안녕"이라고 보내게만 해보는 거예요. 이게 되면 "어, 진짜 되네?" 하는 감이 확 옵니다. 코딩은 역시 AI에게 시켜요.

📍 클로드나 ChatGPT에 그대로 붙여넣기
  1. AI에게 부탁합니다. "내 텔레그램 봇 토큰이랑 내 챗 ID를 넣으면, 텔레그램으로 '안녕! 나 잘 작동해' 라고 한 번 보내는 가장 간단한 파이썬 코드를 줘. 컴퓨터에 파이썬 까는 것부터 실행까지 왕초보용으로 알려줘."
  2. 두 값을 넣고 실행합니다. STEP 2에서 저장해 둔 봇 토큰·챗 ID를 넣고 AI가 시키는 대로 실행 → 텔레그램에 메시지가 '띵' 하고 오면 성공. 축하합니다. 방금 당신은 프로그램으로 스스로에게 메시지를 보낸 거예요.

이 작은 성공이 왜 중요하냐면 — "봇이 나한테 뭔가 보낼 수 있다"가 확인된 순간, 그 '뭔가'에 뭘 넣느냐만 바꾸면 되기 때문입니다. 그럼 이제 재밌는 걸 넣어볼까요?

STEP 4. 뭐든 배달시켜보기 — 따라 하기 좋은 연습 3종

쉬운 순서대로입니다. 하나만 해봐도 감이 확 옵니다. 전부 "AI에게 이렇게 시키세요"만 하면 돼요.

🎯 연습 ① — 매일 아침 뉴스 헤드라인 받기 (제일 쉬움)
"매일 아침, 무료 뉴스 소스에서 오늘 주요 뉴스 제목 5개를 가져와 내 텔레그램으로 보내는 파이썬 코드를 줘. 봇 토큰·챗 ID는 내가 넣을게. 어떤 무료 소스 쓰는지랑 실행법까지 왕초보용으로."

→ 별도 열쇠가 거의 필요 없어서 가장 먼저 해보기 좋아요. '뉴스' 자리에 '오늘 날씨'를 넣으면 아침 날씨 알림이 됩니다.

🎯 연습 ② — 관심 종목·환율·코인 시세 받기
"내가 관심 있는 종목(또는 환율·코인) 시세를 무료로 가져와서, 매일 저녁 텔레그램으로 보내는 코드를 줘. 어떤 무료 소스를 쓰는지랑 실행법도 초보용으로 알려줘."

→ 시세는 소스마다 방식이 조금씩 달라서 뉴스보다 살짝 손이 갑니다. 그래도 AI가 무료 소스까지 골라줘요.

🎯 연습 ③ — 좋아하는 유튜브 채널들 분석하기 (본편으로 가는 다리)
"내가 좋아하는 유튜브 채널 몇 개의 최신 영상과 조회수를 유튜브 API로 가져와, 요즘 어떤 채널이 잘나가는지 텔레그램으로 정리해 보내줘. API 키·봇 토큰·챗 ID는 내가 넣을게. 실행법도 초보용으로."

→ 이때 드디어 STEP 1에서 만든 유튜브 열쇠가 쓰입니다. 여기서 딱 한 걸음만 더 가면 — 남의 채널이 아니라 '내 채널'을 매일 자동으로 챙기는 게 이 시리즈의 본편이에요.

그래서 이 시리즈가 하는 일

눈치채셨나요? 유튜브 채널 자동 관리도 결국 위 연습의 연장선입니다. "창구(API)에서 데이터 가져와 → 우체부(봇)로 나한테 배달"이라는 똑같은 뼈대에, 내 채널·매일 자동 실행·AI 답글을 하나씩 얹어간 거예요. 그 과정을 2편부터 풀어갑니다.

혹시 내 채널 숫자를 자동으로 받고 싶다면, 연습 ③에서 '좋아하는 채널' 대신 내 채널 번호(채널 ID)를 넣으면 됩니다. 채널 ID는 유튜브 스튜디오 › 설정(톱니) › 채널 › 고급 설정 맨 아래에서 복사하세요.

🎁 오늘의 미션 — '내 것'으로 바꿔보기

위 연습 셋 중 딱 하나만 골라 따라 해보세요. 제일 쉬운 ①번(아침 뉴스) 추천. 성공하면 그 순간 당신은 "AI로 뭔가 만들어본 사람"이 됩니다. 막히면 그 화면을 그대로 AI에게 보여주며 "이렇게 나오는데 어떻게 해?"라고 물어보세요 — 그게 바이브코딩입니다.

🕳️ 여기서 삽질했습니다
  • 채널 '이름'으로 찾으니 엉뚱한 채널이 잔뜩. 이름이 비슷한 채널이 세상에 너무 많아서, 남의 채널 숫자가 딸려 왔어요. 그래서 위에서 말한 것처럼 내 채널 고유번호(채널 ID)로 콕 집어야 정확합니다.
  • 구독자 수가 실제와 조금 달랐어요. 알고 보니 유튜브가 구독자를 반올림해서 줍니다(예: 1,847명 → 1,800명). 큰 흐름 보는 데는 문제없지만, "왜 숫자가 딱 안 맞지?" 당황하지 마세요. 원래 그렇습니다.
💡 알아두면 좋은 팁

"어제보다 몇 명 늘었다" 같은 증감은 유튜브가 안 알려줍니다. 그래서 매일 그날 숫자를 파일에 저장해뒀다가 다음 날 값에서 빼는 방식으로 만들었어요(위 AI 부탁에도 이미 넣어놨습니다). 어제의 나를 오늘의 내가 기억하게 하는 것 — 이걸 '스냅샷 비교'라고 부릅니다.

이게 이 연재의 1편이자 가장 쉬운 편입니다. 열쇠 하나, 우체부 하나 만들었을 뿐인데 "내 채널을 매일 챙겨보는 습관"이 자동으로 생겨요. 다음 편에서는 컴퓨터를 꺼놔도 이게 알아서 돌아가게 만듭니다.

세 줄 요약. 1편에서 만든 프로그램은 내 컴퓨터가 켜져 있을 때만 돌아갑니다. 이걸 '깃허브'라는 곳에 올려두면 내 컴퓨터를 꺼놔도 매일 정해진 시간에 알아서 실행돼요. 서버 비용 0원. 이 편도 순서대로 따라만 하면 됩니다.

1편에서 아침 리포트를 만들긴 했는데, 문제가 하나 있었습니다. 제 노트북에 "매일 9시에 실행해라"라고 예약을 걸어놨더니, 정작 아침에 노트북이 꺼져 있으면 아무 일도 안 일어나더라고요. 당연한 얘기죠. 자는 컴퓨터는 일을 못 하니까.

"그럼 항상 켜둬야 하나?" 전기세도 아깝고 현실적이지 않았습니다. 그래서 남의 컴퓨터(정확히는 깃허브의 서버)에 일을 맡기기로 했습니다. 깃허브는 원래 코드를 보관하는 인터넷 창고인데, "정해진 시간에 코드를 대신 실행해주는 기능(깃허브 액션)"이 공짜로 딸려 있어요. 개인 용도로는 한 달에 넉넉히 무료입니다.

STEP 1. 깃허브 가입하고 '창고' 하나 만들기

📍 순서대로 따라 하세요
  1. 깃허브에 가입합니다. 주소창에 github.com → 오른쪽 위 Sign up → 이메일·비밀번호로 가입(무료). 이메일 인증 한 번 하면 끝.
  2. '창고(저장소)'를 하나 만듭니다. 로그인 후 오른쪽 위 + → New repository → 이름 아무거나(예: youtube-alert) → 아래 Private(비공개) 선택 → Create repository. '창고'는 내 코드를 담아두는 인터넷 폴더예요. 여기 올려두면 깃허브가 대신 실행해 줍니다.

STEP 2. 1편 코드를 창고에 올리고, '자동 실행 시간표' 만들기

📍 코드 짜는 건 이번에도 AI가 합니다
  1. 1편에서 AI가 짜준 코드 파일을 창고에 올립니다. 창고 화면에서 Add file → Upload files → 코드 파일(.py)을 끌어다 놓고 → 아래 Commit changes. 끝.
  2. '매일 몇 시에 실행' 시간표 파일을 AI에게 받습니다. AI에게 이렇게 부탁하세요:
    "이 파이썬 코드를 매일 아침 한국시간 9시에 깃허브 액션으로 자동 실행하는 워크플로 파일을 만들어줘. cron은 UTC 기준이니 시차를 반영해서. 초보라서 파일을 어디에 어떻게 올리는지도 알려줘." AI가 파일 내용과 함께 ".github/workflows/ 폴더에 넣으라"고 알려줍니다. 그대로 창고에 올리면 됩니다.

STEP 3. 열쇠·토큰은 '금고'에 따로 넣기 (제일 중요)

📍 코드에 열쇠를 그대로 쓰면 안 됩니다
  1. 왜 금고가 필요할까요? 1편에서 만든 유튜브 열쇠·봇 토큰을 코드에 그대로 적어두면, 그 코드를 보는 사람이 다 볼 수 있어요. 그래서 깃허브가 제공하는 비밀 금고에 따로 넣습니다.
  2. 금고에 값을 넣습니다. 창고 → 위쪽 Settings → 왼쪽 Secrets and variables › Actions → New repository secret → 이름(예: YT_API_KEY)과 값을 넣고 Add secret. 이렇게 유튜브 열쇠 · 봇 토큰 · 내 챗 ID · 채널 ID를 각각 넣어주세요.
  3. 코드가 금고 값을 불러 쓰게 합니다. AI에게 "비밀값들은 코드에 직접 쓰지 말고 깃허브 Secrets에서 불러 쓰게 해줘, 이름은 위와 같이"라고 한마디 하면 알아서 맞춰줍니다.

다 됐으면 창고의 Actions 탭에서 초록색 체크(✔)가 뜨는지 보세요. 초록불이면 성공. 이제 내 컴퓨터를 꺼도 매일 아침 리포트가 옵니다.

🕳️ 여기서 세 번 넘어졌습니다
  • 9시에 걸었는데 밤에 실행됐어요. 제일 흔한 함정입니다. 이 서버는 세계 표준시(영국 시간)로 움직여서, 우리 아침 9시가 거기선 0시예요. 그래서 시간표에 9시라고 쓰면 엉뚱한 시각에 돕니다. → 위 STEP 2에서 "한국시간 9시"라고 콕 집어 AI에게 시키면 시차를 알아서 계산해 줍니다.
  • "올릴 권한이 없다"고 거부당함. 시간표(자동실행) 파일을 올리려는데 막혔어요. 이건 로그인할 때 권한 하나를 더 켜서 다시 인증하면 풀립니다. (웹에서 직접 올리면 잘 안 겪지만, 혹시 이 메시지가 뜨면 이게 원인이에요.)
  • 로그인 승인을 허공에 날렸어요. 인증할 때 화면에 뜬 코드를 승인해야 하는데, 실수로 이미 꺼버린 옛날 창의 코드를 승인해서 아무 일도 안 일어났습니다. → 항상 지금 보고 있는 새 화면의 코드로 승인하세요.
💡 이걸 몰랐으면 두 달 뒤 조용히 멈췄을 팁
  • 깃허브는 60일 동안 아무 활동이 없으면 예약 실행을 스스로 꺼버립니다. 그래서 매일 실행할 때 아주 작은 기록 파일 하나를 같이 남기게 했어요. 그러면 "활동 중"으로 인식돼서 안 꺼집니다.
  • 정확히 정각(9:00)에 걸면 사용자가 몰려서 몇 분씩 밀립니다. 저는 일부러 9시 30분처럼 애매한 시각에 걸어 밀림을 피했어요.

이 편의 핵심 메시지는 하나입니다. "자동화의 진짜 시작은, 내가 없어도 돌아가게 만드는 것." 여기까지 오면 노트북을 덮고 여행을 가도 리포트가 옵니다.

🎁 오늘의 미션 — 내 뉴스봇, 컴퓨터 꺼도 오게

1편에서 만든 아침 뉴스봇, 아직 컴퓨터를 켜놔야만 오죠? 그걸 오늘 배운 대로 깃허브에 올려서 매일 아침 7시, 자는 동안에도 알아서 오게 만들어보세요. 내일 아침 눈뜨자마자 폰에 뉴스가 와 있으면 — 그때 '진짜 자동화'가 뭔지 몸으로 압니다.

3편. 새 댓글·구독자 변동, 1시간 안에 알림 받기 — 유튜브 실시간 감시

세 줄 요약. 유튜브는 새 댓글이 달려도, 영상이 갑자기 떡상해도 먼저 알려주지 않습니다. 그래서 30분마다 스스로 확인하는 '감시병'을 만들었습니다. 변화가 있을 때만 텔레그램으로 콕 집어 알려줘요. 새 프로그램을 또 만드는 게 아니라, 2편에서 만든 걸 늘리는 것이라 더 쉽습니다.

한번은 제 영상 하나가 조회수가 확 튀었는데, 저는 그걸 사흘이나 지나서 알았습니다. 알고리즘을 탄 그 golden time에 댓글도 못 달고, 추가 홍보도 못 하고 그냥 흘려보낸 거죠. 너무 아까웠습니다.

문제는 유튜브가 "야, 너 영상 지금 뜨고 있어!"라고 먼저 알려주지 않는다는 겁니다. 그래서 제가 대신 자주 물어보기로 했어요.

'푸시'가 없으면 '폴링'으로

카톡은 메시지가 오면 알아서 띵 하고 울리죠(이걸 '푸시'라고 합니다). 그런데 유튜브 데이터는 그런 알림을 안 줍니다. 그래서 반대로 갑니다 — 30분에 한 번씩 내가 유튜브한테 "뭐 바뀐 거 있어?"라고 물어보는 방식이에요. 이걸 '폴링(polling)'이라고 부릅니다. 어려운 개념 아니고, 그냥 부지런히 확인 전화 돌리는 거예요.

감시 항목은 네 가지로 정했습니다. ① 새 댓글 ② 구독자 늘고 줄고 ③ 구독자 1,000명 같은 기념비적인 숫자 돌파 ④ 평소보다 확 튀는 급상승. 그리고 중요한 건, 변화가 있을 때만 알림을 보낸다는 겁니다. 아무 일 없는데 30분마다 울리면 그게 스팸이니까요.

따라 하기 — AI에게 '확장'을 시키면 됩니다

새로 뭘 배우는 게 아니에요. 2편에서 만든 걸 조금 늘리는 겁니다. AI에게 아래 두 가지를 부탁하세요.

📍 클로드나 ChatGPT에 그대로 붙여넣기
  1. 감시 기능 추가. "1편·2편 코드를 확장해줘. 30분마다 ① 새 댓글 ② 구독자 변동 ③ 1,000·5,000명 같은 기념 숫자 돌파 ④ 평소보다 급상승한 영상 — 이 네 가지를 확인하고, 변화가 있을 때만 텔레그램으로 알려줘. 아무 변화 없으면 조용히 넘어가고."
  2. 새 영상 24시간 성적표 추가. "새 영상이 올라오면 24시간 뒤에 그 채널 과거 영상들의 중앙값과 비교해서 🚀/😊/😐/🐢로 채점해 보내줘."
  3. 실행 시간표를 30분마다로. 2편에서 만든 것처럼, "이 감시 코드는 30분마다 깃허브 액션으로 실행되게 워크플로 만들어줘"라고 하면 됩니다. 하루 48번 물어봐도 유튜브 무료 한도(하루 10,000회) 안에 넉넉히 들어갑니다.
💡 "급상승"을 어떻게 판단할까

단순히 "조회수 100 넘으면 알림" 식으로 하면, 큰 채널은 맨날 울리고 작은 채널은 영영 안 울립니다. 그래서 "이 영상의 평소 속도보다 몇 배 빠른가"를 기준으로 잡았어요. 최근 흐름을 부드럽게 평균 낸 값과 지금을 비교하는 방식인데(전문 용어로는 EMA), 쉽게 말하면 "얘 오늘따라 유난히 빠르네?"를 숫자로 알아채는 겁니다.

여기에 더해, 새 영상을 올리면 24시간 뒤 자동으로 성적표를 받게도 했습니다. 과거 영상들의 중간값과 비교해서 🚀(대박) / 😊(선방) / 😐(평타) / 🐢(느림)로 채점해줘요. 감으로 "이번 건 좀 되나?" 하던 걸 숫자가 대신 판단해줍니다.

🎁 오늘의 미션 — '떡상 알림' 하나 걸어두기

좋아하는 유튜버가 새 영상을 올리면, 또는 관심 있는 주제의 뉴스가 새로 뜨면 바로 알림이 오게 걸어보세요. 하루 종일 새로고침 안 해도, 변화가 있을 때만 폰이 '띵' 하고 알려줍니다. 이 '변할 때만 알림'이 감시 자동화의 진짜 핵심이에요.

4편. 텔레그램 봇에게 "/현황" 치면 즉시 대답 — 클라우드플레어 워커 입문

세 줄 요약. 아침 리포트를 기다릴 것 없이, 궁금할 때 텔레그램에 "/현황" 한 마디 치면 봇이 바로 채널 상태를 답해줍니다. 24시간 깨어 있어야 하는데, 월 0원짜리 무료 서버(클라우드플레어)로 해결했어요.

아침 리포트가 오는 건 좋은데, 낮에 문득 "지금 구독자 몇이지?"가 궁금할 때가 있잖아요. 그때마다 스튜디오 앱을 열면 또 원점입니다. 그래서 언제든 물어보면 대답하는 비서를 만들기로 했습니다.

24시간 깨어 있으려면 '서버'가 필요한데…

"/현황"이라고 쳤을 때 즉시 답하려면, 그 요청을 받아줄 무언가가 항상 켜져 있어야 합니다. 보통은 서버를 빌리고 매달 돈을 내야 하죠. 여기서 클라우드플레어 워커라는 걸 썼습니다. 이건 서버를 통째로 빌리는 게 아니라 "요청이 올 때만 잠깐 깨어나 일하고 다시 자는" 방식이라(그래서 '서버리스'라고 불러요), 개인 용도로는 사실상 공짜예요.

따라 하기 — 무료 비서 만들기 5단계

📍 순서대로 따라 하세요
  1. 클라우드플레어에 가입합니다. 주소창에 dash.cloudflare.com → 이메일·비밀번호로 가입(무료, 카드 필요 없음).
  2. 워커(내 비서)를 하나 만듭니다. 로그인 후 왼쪽 메뉴 Workers & Pages → Create application → Create Worker → 이름 아무거나 → Deploy. 일단 빈 비서가 하나 생깁니다.
  3. 비서에게 할 일(코드)을 넣습니다. 방금 만든 워커 화면에서 Edit code → AI에게 받은 코드를 붙여넣고 Deploy. AI 부탁 문구: "텔레그램에서 /현황 명령이 오면 유튜브 채널 현황을 답하는 클라우드플레어 워커 코드를 짜줘. 초보라서 어디에 붙여넣는지도 알려줘."
  4. 열쇠·토큰은 여기서도 금고에. 워커 → Settings › Variables and Secrets → 봇 토큰·유튜브 열쇠 등을 Secret으로 추가. (2편의 금고와 같은 개념.)
  5. 텔레그램과 비서를 연결합니다(웹훅). 마지막으로 "텔레그램아, 내 봇 메시지는 이 비서 주소로 보내"라고 한 번 알려줘야 해요. 이걸 '웹훅'이라 합니다. AI가 알려주는 주소를 브라우저 주소창에 한 번 붙여넣고 엔터 → 화면에 "ok":true 가 뜨면 연결 완료. 이제 텔레그램에 /현황을 치면 바로 답이 옵니다.
💡 아무나 못 쓰게 막는 두 가지 잠금장치
  • 비밀 열쇠(시크릿 토큰). 텔레그램과 내 봇 사이에만 아는 암호를 걸어서, 남이 엉뚱하게 명령을 흘려보내도 무시하게 했습니다.
  • 내 채팅만 응답. 봇이 세상 아무한테나 내 채널 숫자를 알려주면 안 되니까, 내 텔레그램 계정에서 온 요청이 아니면 그냥 무응답으로 처리했어요.

이 편은 "자동으로 오는 리포트"에서 "내가 부르면 오는 비서"로 넘어가는 지점입니다. 월 0원인데 24시간 깨어 있는 내 전용 비서가 생긴 거죠. 문과 출신인 저에게도 어렵지 않았으니, 개념만 잡으면 누구나 됩니다.

🎁 오늘의 미션 — 나만의 '/명령어' 만들기

매일 오는 것 말고, 궁금할 때 물어보면 답하는 걸 하나 만들어보세요. /날씨 치면 지금 날씨, /환율 치면 지금 환율 — 뭐든 좋습니다. 텔레그램에 명령어 하나 쳤을 뿐인데 답이 오는 순간, '내 전용 앱'을 가진 기분이 듭니다.

5편. 유튜브 스튜디오급 분석을 자동 리포트로 — 인증(OAuth) 삽질기

세 줄 요약. 조회수 같은 공개 숫자 말고 시청시간·유입경로·수익 같은 진짜 속살 데이터를 자동으로 받으려면, 유튜브에 "이건 내 채널이 맞다"고 로그인 인증을 해줘야 합니다. 이 인증 과정에서 세 번 크게 막혔고, 특히 "7일 뒤 전체가 멈추는 함정"이 있었습니다.

1~3편의 데이터는 '누구나 볼 수 있는' 공개 숫자였습니다. 그런데 시청시간, 어디서 유입됐는지, 수익 같은 건 채널 주인만 볼 수 있죠. 이걸 자동으로 받으려면 유튜브에게 "나 이 채널 주인 맞아"라고 인증(OAuth)을 해줘야 하는데 — 여기가 이 연재에서 제일 험난했습니다. 솔직히 이 편은 좀 까다롭습니다. 대신 아래 순서대로만 하면 삽질 없이 통과할 수 있게 정리했어요.

따라 하기 — 인증 준비 4단계

📍 1편의 그 '구글 클라우드 콘솔'로 다시 갑니다
  1. 분석용 기능을 하나 더 켭니다. console.cloud.google.com 접속(1편에서 만든 프로젝트 선택) → API 및 서비스 › 라이브러리 → YouTube Analytics API 검색 → 사용. (1편에서 켠 것과 별개로 하나 더 켜는 거예요.)
  2. "이 앱을 누가 쓸지" 등록합니다 (제일 중요). API 및 서비스 › OAuth 동의 화면 → 사용자 유형 External → 앱 이름·내 이메일 입력·저장 → '테스트 사용자'에 내 구글 계정을 반드시 추가하세요. 이걸 빼먹으면 아래 첫 번째 삽질("액세스 차단")이 그대로 터집니다.
  3. 로그인 열쇠(OAuth 클라이언트)를 만듭니다. API 및 서비스 › 사용자 인증 정보 → + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID → 유형은 데스크톱 앱 → 만들기 → JSON 파일 다운로드. 이 파일이 로그인 열쇠입니다. 잘 보관하세요.
  4. AI에게 코드를 시키고, 한 번 로그인합니다. "이 OAuth 클라이언트 JSON 파일로 유튜브 애널리틱스에 로그인해서, 시청시간·유입경로·수익을 받아 주간/월간 리포트를 텔레그램으로 보내는 코드를 짜줘. 초보라서 로그인 절차도 알려줘." 처음 실행하면 브라우저가 뜨고 내 계정으로 로그인하는데, 여기서 아래 세 관문이 나옵니다.
참고: 구글이 최근 이 메뉴 이름을 'Google 인증 플랫폼'으로 바꿔서, 화면 문구가 위와 조금 다를 수 있습니다. 하지만 하는 일(테스트 사용자 등록 → 클라이언트 만들기 → 게시 전환)은 똑같아요.

세 관문 — 이 화면이 뜨면 이렇게 하세요

🕳️ 삽질 3연타
  • "액세스가 차단되었습니다". 처음엔 내가 만든 프로그램인데도 로그인이 막혔어요. 원인은 "이 앱을 쓸 사람 명단에 나를 안 넣어서"였습니다. 테스트 사용자로 내 계정을 등록하니 풀렸어요.
  • "확인되지 않은 앱"이라는 무서운 경고. 빨간 경고 화면이 떠서 겁먹었는데, 이건 구글 심사를 안 받은 개인 앱이면 다 뜨는 정상 경고였습니다. '고급 → 계속 이동'을 누르면 통과됩니다.
  • 일주일 뒤 시스템 전체가 죽는 함정. 이게 제일 무서웠어요. 테스트 상태로 받은 인증은 딱 7일 만에 만료됩니다. 이걸 모르면 잘 돌던 자동화가 일주일 뒤 조용히 전멸해요. → 해결은 OAuth 동의 화면으로 다시 가서 상태를 앱 게시(프로덕션으로 전환) 로 바꾸면 됩니다. 그러면 만료가 풀려요. 이 한 줄 몰라서 며칠 헤맬 수 있으니 꼭 기억하세요.
💡 헛수고 방지 팁
  • CTR(클릭률)과 노출수는 이 방법으로 못 가져옵니다. 그건 유튜브 스튜디오 화면 전용이라, 아무리 코드를 짜도 안 나와요. 저는 이걸 모르고 한참 시도하다 포기했습니다. 여러분은 처음부터 빼세요.
  • 채널이 여러 개면(브랜드 계정), 채널마다 인증을 따로 해줘야 합니다. 하나 해놓고 다 될 줄 알았다가 또 한 번 놀랐어요.

고생한 만큼, 이 편이 검색으로 사람들이 제일 많이 찾아올 글이라고 봅니다. "유튜브 API 액세스 차단" 같은 걸로 헤매는 사람이 저 말고도 분명 많을 테니까요. 삽질이 곧 콘텐츠가 되는 순간입니다.

🎁 오늘의 미션 — 일요일 밤, 내 채널 주간 성적표

제일 어려운 산을 넘었으니 이제 열매를 따야죠. 내 채널 지난 한 주 시청시간(또는 수익)을 일요일 밤마다 자동으로 받아보게 만들어보세요. 한 주를 숫자로 돌아보는 습관이, 다음 주에 올릴 영상을 바꿔놓습니다.

6편. AI가 내 유튜브 댓글에 답글을 달아준다 — 승인 버튼 하나로

세 줄 요약. 시청자 댓글이 달리면 AI가 답글 초안을 써서 텔레그램으로 보내줍니다. 저는 [게시] 버튼만 한 번 누르면 그대로 유튜브에 올라가요. 완전 자동이 아니라 '사람이 마지막에 승인'하는 방식 — 이게 핵심입니다.

여기까지 오니 욕심이 났습니다. "댓글 답글도 AI가 달아주면 안 되나?" 그런데 막상 완전 자동으로 답글이 나가게 하려니 무섭더라고요. AI가 엉뚱하거나 무례한 답을 달면 그대로 제 채널에 박제되니까요.

그래서 타협점을 찾았습니다. AI는 초안까지만, 최종 게시는 사람이. 댓글이 달리면 AI가 답글을 써서 텔레그램으로 "이렇게 달까요?"라고 물어봐요. 저는 [게시] 또는 [무시] 버튼만 누르면 됩니다. 탭 한 번으로 끝.

자동화의 수위를 정하는 문제

이 편에서 진짜 하고 싶은 이야기는 기술이 아니라 "어디까지 기계에 맡길 것인가"입니다. 편하자고 다 맡기면 사고가 나고, 다 직접 하면 자동화한 의미가 없죠. "AI가 90%, 내가 마지막 10%" — 이 지점을 찾는 게 바이브코딩의 진짜 실력이라고 느꼈습니다.

따라 하기 — AI 열쇠 받고 붙이기

📍 답글을 써줄 AI(OpenAI)의 열쇠부터 만듭니다
  1. OpenAI 열쇠(API 키)를 발급합니다. 주소창에 platform.openai.com → 로그인/가입 → 오른쪽 위 프로필 → API keys → Create new secret key → 뜨는 열쇠를 바로 복사해 저장(이 화면을 닫으면 다시 못 봅니다). ⚠️ 답글 쓸 때 아주 적은 돈이 드니 결제수단 등록 + 소액 충전이 필요해요. 저는 $10 넣고 아직도 씁니다.
  2. AI에게 답글 시스템을 시킵니다. "유튜브에 새 댓글이 달리면 OpenAI로 답글 초안을 만들고, 텔레그램으로 [게시] [무시] 버튼과 함께 보내줘. 내가 [게시]를 누르면 그 답글이 유튜브에 올라가게. 이건 2편 깃허브 액션 + 4편 클라우드플레어 워커 위에 얹어줘."
  3. OpenAI 열쇠도 금고에 넣습니다. 2·4편에서 했던 것처럼, 이 열쇠도 코드에 직접 쓰지 말고 깃허브·클라우드플레어 금고(Secrets)에 넣어 불러 쓰게 하세요.
🕳️ 여기서 삽질했습니다
  • AI가 자꾸 빈 답을 뱉었어요. 분명 요청했는데 답글이 텅 비어서 왔습니다. 알고 보니 AI가 속으로 생각(추론)하는 데 정해진 분량을 다 써버려서 정작 내놓을 답이 없던 거였어요. "너무 깊게 고민하지 말고 바로 답해"라고 설정을 바꾸니 해결됐습니다.
  • 댓글 '고정'은 자동이 안 됩니다. 답글을 다는 것까지는 자동인데, 그 댓글을 맨 위에 고정하는 건 유튜브가 자동으로 못 하게 막아놨어요. 그건 어쩔 수 없이 손으로 합니다.
💡 소소하지만 중요한 예외 처리
  • 스팸 댓글엔 답 안 달기. AI에게 "광고·스팸이면 그냥 SKIP만 답해"라고 시켜서, 쓸데없는 답글이 나가지 않게 걸렀습니다.
  • 내가 단 댓글엔 답 안 하기. 안 그러면 AI가 제 댓글에 또 답글을 달고, 거기 또 답하고… 무한 반복이 될 뻔했어요. 그래서 "채널 주인 댓글이면 건너뛰기" 예외를 넣었습니다.

이렇게 6편이 끝납니다. 아침 리포트로 시작해서, 컴퓨터를 꺼도 돌아가게 만들고, 실시간 감시와 즉답 봇을 붙이고, 속살 데이터까지 끌어오고, 마지막엔 AI가 답글까지 — 전부 무료로, 하루 만에. 개발자가 아니어도, AI와 함께라면 여기까지 옵니다. 저는 이걸 "AI로 양산"이 아니라 "AI로 증폭"이라고 부르고 싶어요.

🎁 오늘의 미션 — AI에게 '내 말투' 입히기

AI가 쓴 답글이 조금 남 같나요? 그럼 내가 실제로 단 답글 2~3개를 예시로 보여주며 "이런 말투로 써줘"라고 해보세요. 신기하게 나처럼 씁니다. AI가 90%, 내가 10% — 그 10%에 '나다움'을 넣는 게 이 시리즈의 진짜 결론이에요.
